# Requirements: Frontend Admin Panel

## Feature Overview

Минималистичная, современная веб-панель управления для владельца компании.
Позволяет настраивать подписки на уведомления о тендерах: задавать фильтры поиска и список email-получателей.
Доступ — через HTTP Basic Auth.

---

## User Stories

### US-1: Вход в панель
**Как** владелец компании
**Я хочу** войти в панель по логину и паролю
**Чтобы** никто посторонний не мог изменить мои настройки

**Acceptance Criteria:**
- При переходе на `/admin` браузер запрашивает Basic Auth credentials
- Неверные credentials → 401, браузер повторно просит войти
- После успешного входа открывается главная страница панели

---

### US-2: Просмотр подписок
**Как** владелец
**Я хочу** видеть список всех подписок
**Чтобы** понимать, на что настроены уведомления

**Acceptance Criteria:**
- Страница `/admin/subscriptions` показывает таблицу подписок
- Каждая строка: название, источник (реестр), список email, статус (активна/нет)
- Если подписок нет — пустой стейт с кнопкой "Создать первую"

---

### US-3: Создание подписки
**Как** владелец
**Я хочу** создать новую подписку
**Чтобы** получать уведомления о нужных тендерах на нужные почты

**Acceptance Criteria:**
- Кнопка "Новая подписка" открывает форму (страница или drawer)
- Поля формы:
  - **Название** — произвольная строка (обязательно)
  - **Источник** — выпадающий список реестров (например: gosplan)
  - **Фильтры** — секция с полями фильтрации тендеров (ключевые слова, регион, статус, сумма от/до)
  - **Email-список** — текстовое поле, несколько адресов через запятую или каждый с новой строки
- Валидация: название обязательно, хотя бы один email, email-адреса корректны
- После сохранения — возврат к списку, новая подписка появляется в таблице

---

### US-4: Редактирование подписки
**Как** владелец
**Я хочу** изменить фильтры или список email существующей подписки
**Чтобы** скорректировать уведомления без пересоздания

**Acceptance Criteria:**
- Клик по строке или кнопка "Редактировать" открывает форму с заполненными данными
- Источник (source) нельзя изменить — поле отображается как read-only
- Все остальные поля редактируемы
- После сохранения — данные обновляются в таблице

---

### US-5: Удаление подписки
**Как** владелец
**Я хочу** удалить ненужную подписку
**Чтобы** не получать лишние уведомления

**Acceptance Criteria:**
- Кнопка "Удалить" в строке таблицы показывает диалог подтверждения
- После подтверждения подписка исчезает из списка

---

### US-6: Включение/выключение подписки
**Как** владелец
**Я хочу** временно приостановить подписку без удаления
**Чтобы** не потерять настройки

**Acceptance Criteria:**
- Toggle (переключатель) в строке таблицы меняет статус активности
- Неактивная подписка визуально отличается (приглушённый цвет)

---

## Functional Requirements

### P0 (обязательно к релизу)
- **F-01**: Basic Auth через Spring Security — защита всех `/admin/*` маршрутов
- **F-02**: Список подписок с пагинацией (если > 20)
- **F-03**: Создание подписки с валидацией формы
- **F-04**: Редактирование подписки (фильтры + email-список)
- **F-05**: Удаление подписки с подтверждением
- **F-06**: Фронт собирается как static и раздаётся Spring Boot из `/admin`

### P1 (желательно)
- **F-07**: Включение/выключение подписки через toggle
- **F-08**: Поиск/фильтрация по названию в таблице подписок
- **F-09**: Toast-уведомления об успехе/ошибке операций
- **F-10**: Индикатор загрузки (skeleton / spinner) при запросах

### P2 (опционально)
- **F-11**: Страница со списком последних найденных тендеров по каждой подписке
- **F-12**: Ручной запуск проверки тендеров (кнопка "Проверить сейчас")

---

## Non-Functional Requirements

- **NFR-01**: SPA на React 18 + TypeScript + Vite
- **NFR-02**: UI-компоненты — shadcn/ui (Radix UI + Tailwind CSS)
- **NFR-03**: HTTP-клиент — fetch / TanStack Query для кэширования и состояния
- **NFR-04**: Фронт собирается командой `npm run build` в `src/main/resources/static/admin`
- **NFR-05**: Все запросы к API идут на тот же origin (`/api/v1/*`), Basic Auth передаётся браузером автоматически
- **NFR-06**: Поддержка светлой и тёмной темы — переключатель в шапке, выбор сохраняется в localStorage
- **NFR-07**: Адаптивность — корректное отображение на 1280px+ (десктоп, целевая аудитория — один человек)
- **NFR-08**: Время первой загрузки страницы — < 2s на хорошем интернете

---

## Constraints & Assumptions

- Один пользователь (владелец) — multi-user не нужен
- Credentials хранятся в `application.yml` (spring.security), не в БД
- Фронт-проект живёт в `frontend/` в корне репозитория
- API уже реализован (`/api/v1/subscriptions`) — фронт только потребляет его
- Браузерный Basic Auth диалог достаточен — отдельная страница логина не нужна

---

## Out of Scope

- Регистрация / управление несколькими пользователями
- Мобильная версия
- Отдельный фронтенд-сервер или CDN
- Интернационализация (только RU)

---

## Success Metrics

- Владелец может создать подписку за < 2 минут без инструкций
- Все P0 требования реализованы и работают без ошибок в консоли
- Фронт успешно собирается и раздаётся Spring Boot одной командой `./gradlew build`
