# Requirements: Local Keywords API

## Overview

Поле `filterLocalKeywords` уже существует в БД и используется для пост-фильтрации
тендеров по `objectInfo` (после получения результатов от API закупок).
Сейчас оно не доступно через REST API и не отображается в UI — задать его можно только
вручную через SQL. Нужно пробросить поле полностью: бэкенд → API → фронтенд.

## User Stories

### US-1: Задать локальные ключевые слова через UI
**Как** администратор подписки,
**я хочу** указать список слов для вторичной фильтрации,
**чтобы** из результатов API оставались только тендеры, реально связанные с нужной категорией.

**Критерии приёмки:**
- В форме создания/редактирования подписки есть поле «Локальные ключевые слова»
- Поле принимает список слов в виде тегов (как keywords)
- Слова сохраняются в БД и возвращаются при GET подписки
- При пустом списке фильтр не применяется (все тендеры проходят)

### US-2: Видеть локальные ключи в списке подписок
**Как** администратор,
**я хочу** видеть текущие локальные ключи у каждой подписки,
**чтобы** понимать что именно фильтруется.

**Критерии приёмки:**
- `localKeywords` возвращается в `SubscriptionResponse.filters`

## Functional Requirements

### P0 (обязательно)

- **FR-1** `SubscriptionFiltersRequest` содержит поле `localKeywords: List<String>` (default `emptyList()`)
- **FR-2** `SubscriptionRequest.toEntity()` сохраняет `localKeywords` как CSV в `filterLocalKeywords`
- **FR-3** `Subscription.toResponse()` включает `localKeywords` из `filterLocalKeywords`
- **FR-4** `SubscriptionUpdateRequest` тоже поддерживает `localKeywords`
- **FR-5** Фронтенд: тип `SubscriptionFilters` содержит `localKeywords: string[]`
- **FR-6** Фронтенд: форма содержит TagInput «Локальные ключевые слова» внутри блока «Доп. фильтры»
- **FR-7** Фронтенд: при создании/редактировании `localKeywords` передаётся в payload

### P1 (важно)

- **FR-8** Тест `TenderLocalKeywordFilterSpec.householdGoodsFilters()` обновлён расширенным списком из 27 слов
- **FR-9** В UI есть подсказка что это вторичный фильтр по названию тендера

### P2 (желательно)

- **FR-10** Предзаполненные «стартовые» слова для категории «хоз товары» доступны одной кнопкой

## Non-functional Requirements

- Изменения backward-compatible: подписки без `localKeywords` продолжают работать (поле nullable)
- Нет новых миграций БД (колонка уже существует)
- Фронтенд-изменения не ломают существующие тесты

## Out of Scope

- Управление пресетами ключевых слов по категориям
- Автодополнение / предложение слов
- Отдельный эндпоинт только для обновления `localKeywords`

## Success Metrics

- Все тесты `TenderLocalKeywordFilterSpec` зелёные ✓ (уже выполнено)
- Поле `localKeywords` присутствует в ответе GET `/api/v1/subscriptions`
- Поле редактируемо через UI без прямого доступа к БД
